Our Principles

Extract Text from Image has designed this policy to be consistent with the following principles:

  • Privacy policies should be human readable and easy to find.
  • Data collection, storage, and processing should be simplified as much as possible to enhance security, ensure consistency, and make the practices easy for users to understand.
  • Data practices should meet the reasonable expectations of users.
The extension does not collect personal information, does not require an account, does not use cloud servers for OCR processing, and does not include any analytics or tracking. All text recognition runs locally inside your browser.

Information We Collect

The extension is designed to operate without collecting personally identifiable information. We collect information in two ways: information generated locally on your device during use, and information transmitted only as a standard part of one-time public network requests required to deliver the Services.

Information Stored Locally on Your Device

The extension stores the following data exclusively in your browser using the Chrome storage.local API. This data never leaves your device and is not transmitted to us or any third party:

  • Selected OCR language — the language model you choose in the extension popup (for example, English, German, Russian, Japanese). Stored as a short string so the extension remembers your preference between sessions.
  • Cached OCR model files — the machine-learning model files used for text recognition are cached by your browser after the first download, so subsequent recognitions work offline. Approximately 15 MB total per language.

Image Data and Recognized Text

When you initiate a recognition by clicking the extension icon and selecting a region on the current page, a screenshot of that region is captured using Chrome's built-in tabs.captureVisibleTab API. The screenshot is processed entirely inside your browser by a local PaddleOCR WebAssembly engine running in an offscreen document. No image data, no recognized text, and no associated metadata are transmitted to us or to any external server at any point. The recognized text is displayed in a panel inside the active page and remains there only until you close the panel or navigate away.

One-Time Model Download

On first use of a given language, the extension downloads the corresponding OCR model files (approximately 15 MB) from a public GitHub repository operated by PT Perkasa Pilar Utama:

  • github.com/PT-Perkasa-Pilar-Utama/ppu-paddle-ocr-models

This download is performed by your browser as a standard HTTPS request, identical to loading any public webpage. GitHub may log the request (including your IP address) as part of standard server operations governed by GitHub's own privacy policy. After download, model files are cached locally by your browser and reused for all subsequent recognitions without further network requests.

Security Measures

We have designed the extension to minimize the attack surface for user privacy: no backend servers operated by us, no third-party SDKs, no analytics, no advertising, and no transmission of user content. The only network request is the HTTPS download of public model files from GitHub. The extension's request for broad host permissions (all_urls) is used solely to allow you to activate text recognition on any webpage of your choosing — no page content is ever read, captured, or transmitted unless you explicitly click the extension icon and draw a selection. However, we cannot and do not guarantee that these measures will prevent every possible privacy or security issue, as no Internet-connected software can be completely secure.
